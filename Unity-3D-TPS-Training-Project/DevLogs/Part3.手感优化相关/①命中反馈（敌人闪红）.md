## 命中反馈

目标：被打中 → 敌人闪红 0.1 秒 → 恢复

不是“逻辑需求”，而是“感知需求”。

### 变量部分

```
private Renderer enemyRenderer;
private Color originalColor;
```
Renderer是什么？

负责显示物体外观的组件，我的player和enemy有颜色是因为

身上有Mesh Renderer

OriginalColor存什么？

存原始颜色

目标是：改颜色（受击）再改回去

Start（）里两句
```
enemyRenderer = GetComponent<Renderer>();
originalColor = enemyRenderer.material.color;
```

由于没有xxx.GetComponent，于是直接是从挂载的敌人自己身上找组件

enemyRenderer.material.color表示enemy的颜色

TakeDamage中的新知识点

StartCoroutine(FlashRed());

Coroutine 是什么？

普通函数：开始 → 跑完 → 结束


协程函数：开始 → 跑一部分 → 暂停 → 等待 → 继续跑

StartCoroutine 的作用

StartCoroutine() 是启动协程的方法，它让一个函数可以：

暂停执行（等待一段时间或条件）

分帧执行（把耗时分到多帧）

保持状态（暂停后恢复时保持变量值）

关于FlashRed函数

被协程函数调用了

第一步
IEnumerator FlashRed()

是指 协程的固定返回类型

意味着这个函数可以**暂停**

第二步
enemyRenderer.material.color = Color.red;
直接把敌人变红

第三步

yield return new WaitForSeconds(0.1f);

意思：

“暂停这个函数 0.1 秒”

不是暂停游戏，而是暂停协程

时间线：

os（受伤变红）->0-0.1s(协程等待)->0.1s(变回原来的颜色)
