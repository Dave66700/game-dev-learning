构造函数到底是什么？

类比：

类（class）：“人类”这个设计蓝图。每个人都应该有，名字、年龄、说话功能等等。

对象（Object）：根据蓝图制造出来的具体的人，张三。

构造函数（Constructor）：就是在“张三”这个对象诞生的那一刻，**自动执行**的一个初始化过程。这个过程确保“张三”一出生，就有了一些基本的属性。

“出生过程”
- 自动执行，“张三”一出生，构造函数就自动运行了
- 通常用来初始化，给“张三”起名字、测身高体重，具备初始状态
- 名字和类型名一样，“出生过程”需要和“人类”蓝图名字一模一样。才知道初始化的是人类。

- 代码举例：
```
  public class Human
{
    public string name;
    public int age;

    // 这就是构造函数，名字和类名一样，没有返回值
    public Human()
    {
        name = "New Born Baby"; // 初始化名字
        age = 0;                // 初始化年龄
        Debug.Log("一个人类诞生了！");
    }
}
```
在其他地方 new Human() 的时候，Human 里的那段初始化代码就会自动运行，打印出日志，并设置好名字和年龄。

我的FSM

```
using UnityEngine;

// 所有敌人状态的父类
public abstract class EnemyState
{
    protected EnemyAI enemy;

    // 构造函数：让每个状态都知道自己属于哪个敌人
    public EnemyState(EnemyAI enemy)
    {
        this.enemy = enemy;
    }

    // 进入状态时调用
    public virtual void Enter() { }

    // 每帧执行
    public virtual void Update() { }

    // 离开状态时调用
    public virtual void Exit() { }

```




EnemyState继承的是object 不是MonoBehaviour，从而不需要挂载到GameObject上面。

作用纯粹，就是**建立关联**。把状态和所属的敌人关联起来

EnemyAI 是挂载在敌人身上的 MonoBehaviour，而它的各种状态（IdleState, ChaseState等）是辅助工作的普通C#对象。
